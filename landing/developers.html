Getting Started

Welcome to the Kerisi Lite API! This guide will help you get started with integrating our financial management capabilities into your application.

Base URL: https://api.kerisilite.com/v1

Quick Start

  1. Create an account at Kerisi Lite Dashboard
  2. Generate your API key from Settings → Integrations
  3. Make your first API call using the examples below

Example Request

curl https://api.kerisilite.com/v1/invoices \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"

Authentication

The Kerisi Lite API uses API keys to authenticate requests. Include your API key in the Authorization header of every request.

Getting Your API Key

  1. Log in to your Kerisi Lite dashboard
  2. Navigate to Settings → Business Settings → Integrations
  3. Click "Generate New API Key"
  4. Copy and securely store your API key

Warning: Keep your API keys secure! Do not share them in publicly accessible areas such as GitHub, client-side code, etc.

Authentication Header

Authorization: Bearer YOUR_API_KEY

API Endpoints

Invoices

GET /invoices

Retrieve a list of all invoices

fetch('https://api.kerisilite.com/v1/invoices', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
})
.then(response => response.json())
.then(data => console.log(data));
POST /invoices

Create a new invoice

fetch('https://api.kerisilite.com/v1/invoices', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    customer_id: 123,
    invoice_number: 'INV-2025-001',
    invoice_date: '2025-12-29',
    due_date: '2026-01-28',
    items: [
      {
        description: 'Website Development',
        quantity: 1,
        rate: 8500.00
      }
    ],
    tax_rate: 6,
    notes: 'Thank you for your business'
  })
})
.then(response => response.json())
.then(data => console.log(data));
GET /invoices/:id

Retrieve a specific invoice

curl https://api.kerisilite.com/v1/invoices/12345 \
  -H "Authorization: Bearer YOUR_API_KEY"
PUT /invoices/:id

Update an existing invoice

DELETE /invoices/:id

Delete an invoice

Webhooks

Webhooks allow you to receive real-time notifications when events occur in your Kerisi Lite account.

Available Events

  • invoice.created

    Triggered when a new invoice is created

  • invoice.paid

    Triggered when an invoice is marked as paid

  • customer.created

    Triggered when a new customer is added

  • expense.created

    Triggered when a new expense is recorded

Webhook Payload Example

{
  "event": "invoice.created",
  "timestamp": "2025-12-29T10:30:00Z",
  "data": {
    "id": 12345,
    "invoice_number": "INV-2025-001",
    "customer_id": 123,
    "total": 9010.00,
    "status": "pending"
  }
}

SDKs & Libraries

Official SDKs to integrate Kerisi Lite API in your favorite programming language.

JavaScript / Node.js

npm install kerisi-lite

const Kerisi = require('kerisi-lite');
const client = new Kerisi('YOUR_API_KEY');

const invoices = await client.invoices.list();

Python

pip install kerisi-lite

import kerisi

client = kerisi.Client('YOUR_API_KEY')
invoices = client.invoices.list()

PHP

composer require kerisi/lite

$client = new \Kerisi\Client('YOUR_API_KEY');
$invoices = $client->invoices->list();

Ruby

gem install kerisi-lite

client = Kerisi::Client.new('YOUR_API_KEY')
invoices = client.invoices.list

Rate Limits

To ensure fair usage and system stability, API requests are rate limited.

Plan Rate Limit Burst
Starter 100 requests/minute 200 requests
Professional 1,000 requests/minute 2,000 requests
Enterprise Custom Custom

Rate limit headers are included in every API response: X-RateLimit-Limit, X-RateLimit-Remaining

Error Codes

The Kerisi Lite API uses conventional HTTP response codes to indicate success or failure.

200 - OK

Everything worked as expected

400 - Bad Request

The request was unacceptable, often due to missing a required parameter

401 - Unauthorized

No valid API key provided

404 - Not Found

The requested resource doesn't exist

429 - Too Many Requests

Too many requests hit the API too quickly

500, 502, 503, 504 - Server Errors

Something went wrong on Kerisi Lite's end

Error Response Format

{
  "error": {
    "type": "invalid_request_error",
    "message": "Invalid customer_id provided",
    "param": "customer_id",
    "code": "invalid_parameter"
  }
}
